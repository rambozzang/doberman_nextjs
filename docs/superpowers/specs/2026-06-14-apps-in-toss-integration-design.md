# 도배르만 × 앱인토스(Apps in Toss) 입점 설계 문서

- 작성일: 2026-06-14
- 대상 서비스: 도배르만 웹 (도배 견적 플랫폼, Next.js 16 App Router)
- 목표: 토스 앱 안에서 "앱인앱(App-in-App)" 형태로 도배 견적 서비스 제공

---

## 1. 배경 & 목표

토스의 **앱인토스(Apps in Toss)** 플랫폼에 도배르만 견적 서비스를 입점시킨다. 토스 앱 안에서 별도 설치 없이 파트너 서비스를 제공하는 App-in-App 구조이며, 토스 약 3,000만 유저에게 노출된다.

### 전략 요약

| 항목 | 결정 |
|---|---|
| 플랫폼 | 앱인토스 (토스 내 App-in-App) |
| 연동 방식 | **WebView** — 기존 Next.js 사이트를 그대로 임베드 |
| 핵심 타깃 | B2C 고객, **AI 견적(`/quote-request-ai`)을 메인 후킹 포인트** |
| 사업자 | 보유(법인/개인사업자) → 검수·토스로그인·수익화 모두 가능 |
| 일정 | 1~2개월 균형형, 단계적 출시 |
| 차별점 | "토스에서 30초 만에 AI로 도배 견적 받기" |

### 앱인토스 핵심 사실 (개발자센터 기준, 2026-06)

- 연동 방식: WebView 또는 React Native SDK 둘 다 지원 → **본 프로젝트는 WebView 채택**
- 기존 운영 서비스 재활용 가능 (클라이언트만 토스 환경에 맞게 조정 후 자체 백엔드 연동)
- 토스 로그인/결제는 필수 아님. 단 **사업자 등록 시** 토스 로그인·토스페이·인앱결제·광고 사용 가능
- 출시 절차: 콘솔 가입 → 워크스페이스 생성 → 사업자 등록 → 앱 등록 → 검수 요청
- 검수: 서비스/기능/디자인/보안 검토, 영업일 3~5일
- 비게임 서비스 개발 기간: 1~3개월
- 요건: 만 19세+ 대상, Android 7+ / iOS 16+
- 수수료: 인앱결제 시 앱마켓 15~30% + 토스 5% / 광고 토스 15% / 정산 익월 말일

## 2. 현재 시스템 (재활용 자산)

- **프레임워크**: Next.js 16.1.0 (App Router), React 19
- **상태/데이터**: Zustand, TanStack React Query, Axios
- **스타일**: Tailwind CSS 3.4 (반응형), PWA(manifest) 지원
- **인증**: JWT(localStorage `auth_token`/`user_info`) + Axios 인터셉터(Bearer 자동주입, 401 리다이렉트)
- **소셜로그인**: 구글(`/auth/google/callback`), 카카오(`/auth/kakao`), 네이버(`/auth/naver`) + 휴대폰 인증
- **백엔드**: 별도 서버 `https://www.tigerbk.com/api-doman/web` 와 REST 연동, 채팅은 WebSocket
- **핵심 B2C 플로우**: AI 견적(`/quote-request-ai`), 일반 견적(`/quote-request`), 견적 계산기, 내 견적 조회
- **결제**: RevenueCat 기반 구독(사장님 B2B `/boss/billing`)

→ 반응형·PWA·JWT·소셜로그인이 이미 갖춰져 있어 WebView 입점에 기술적으로 유리.

---

## 3. 단계적 로드맵

### Phase 1 — WebView 입점 (1차 출시, ~4주)
- 기존 웹을 앱인토스 WebView로 그대로 띄움
- 토스 진입 전용 경량 라우트(`/toss/...`) — 헤더/푸터 등 불필요 UI 제거
- AI 견적 → 견적요청 → 결과확인 핵심 플로우만 노출
- 기존 인증·결제 그대로 유지
- **목표**: 검수 1회 통과 및 출시

### Phase 2 — 토스 네이티브 기능 통합 (2차, ~4주)
- 토스 로그인 SDK 연동 (기존 소셜로그인과 병행, 자체 JWT 발급 연계)
- 토스 환경 감지 → 브릿지(뒤로가기/공유/스크롤 등) 대응
- 푸시·딥링크, 토스 디자인 가이드 일부 적용

### Phase 3 — 수익화 (선택, 추후)
- 토스페이/인앱결제 vs 기존 RevenueCat 수수료 비교 후 결정 (사장님 B2B용)
- 본 문서 범위 밖. 별도 스펙으로 분리.

---

## 4. Phase 1 상세 설계 (이번 구현 범위)

### 4.1 토스 전용 진입점 & 경량 레이아웃
- 신규 라우트 그룹: `app/toss/` (예: `/toss` 랜딩, `/toss/quote` AI 견적 진입)
- 토스 환경에 맞춘 **경량 레이아웃**: 글로벌 헤더/푸터/사이드 내비 제거, 콘텐츠 풀스크린, safe-area 패딩
- 진입 시 AI 견적 후킹을 전면에 배치 ("토스에서 30초 만에 AI 도배 견적")

### 4.2 환경 감지 유틸
- "현재 토스 WebView 안인지" 판별하는 유틸(`lib/toss/env.ts` 등): UserAgent 패턴 + 토스 브릿지 객체(window 주입) 존재 여부
- 결과로 UI 분기(토스 모드면 경량 레이아웃/외부링크 처리 변경)

### 4.3 WebView 호환성 처리
- viewport / safe-area(notch) 대응
- 외부 링크: 새창(`target=_blank`) 대신 토스 브릿지 또는 동일 WebView 내 처리로 변경
- iOS 스크롤 바운스/오버스크롤 정리
- 카메라/사진 업로드(현장사진·포트폴리오) 권한 동작 검증

### 4.4 인증 흐름 점검 (최우선 검증)
- WebView 내 localStorage/쿠키 정상 동작 확인
- **소셜로그인 OAuth 팝업·리다이렉트가 WebView에서 차단되는지 검증** — 차단 시 대체안:
  1. 휴대폰 인증 로그인을 토스 환경 기본값으로
  2. 또는 토스 로그인(Phase 2)을 앞당김
- 비로그인으로도 AI 견적 시작 가능하도록(전환율 보호), 결과 저장 시점에만 로그인 유도

### 4.5 앱인토스 콘솔 셋업 (개발 외 운영 작업)
- 콘솔 가입 → 워크스페이스 생성 → 사업자 등록 → 앱 등록 → 검수 요청
- 검수 대비 체크리스트: HTTPS, 개인정보 처리방침, 19세+ 정책, iOS16+/Android7+ 동작

---

## 5. 리스크 & 검증 포인트

| 리스크 | 대응 |
|---|---|
| 소셜로그인 OAuth 팝업/리다이렉트가 WebView에서 차단 | Phase 1 최우선 검증. 막히면 휴대폰 인증 기본화 또는 토스 로그인 조기 도입 |
| 카메라/사진 업로드 WebView 권한 | 토스 WebView 권한 정책 확인 후 분기 처리 |
| 외부 결제/외부링크 정책 위반으로 검수 리젝 | B2B 결제는 Phase 3로 분리, 외부링크는 브릿지 처리 |
| 검수 리젝에 따른 3~5영업일 지연 | 사전 체크리스트로 1회 통과 목표 |

## 6. 성공 지표 (KPI)
- 토스 유입 → AI 견적 시작률
- 견적요청 완료율(퍼널 전환)
- 토스 경유 신규 고객 수 / 사장님 매칭 건수

---

## 7. 범위 (Scope)

### 포함 (Phase 1)
- 토스 전용 경량 진입 라우트 및 레이아웃
- 토스 WebView 환경 감지 유틸
- WebView 호환성 처리 (외부링크/스크롤/권한/safe-area)
- 인증 흐름 WebView 검증 및 대체안
- 검수 대비 점검

### 제외 (후속 Phase)
- 토스 로그인 SDK 연동 (Phase 2)
- 토스 브릿지 기반 푸시/딥링크/공유 (Phase 2)
- 토스페이/인앱결제 등 수익화 (Phase 3)
- B2B 사장님 기능의 토스 노출 (현 단계 B2C 우선)
