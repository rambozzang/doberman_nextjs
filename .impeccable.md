## Design Context

### Users
- **Primary users**: Wallpaper installation business owners (도배사장님) and their office staff in South Korea.
- **Context**: Used mainly on desktop/PC in the office or on-site with a laptop. The app is a work tool for managing customer quotes, orders, schedules, construction records, AS requests, portfolio, community, and subscriptions.
- **Jobs to be done**: Quickly find the next task, manage customer pipeline, create estimates/checklists, track construction progress, handle customer complaints (AS), view sales stats, and manage billing.
- **Emotional goals**: Confidence (data is reliable), trust (professional look), calm (no visual noise), efficiency (dense information, fast scanning).

### Brand Personality
- **3 words**: Professional, Trustworthy, Concise.
- **Tone**: Straightforward, business-like, supportive. Avoid playful or overly decorative language.
- **Voice**: Use clear labels, action-oriented button copy, and helpful empty/error states.

### Aesthetic Direction
- **Visual tone**: Clean B2B admin/SaaS dashboard. Sharp hierarchy, generous whitespace on desktop, information-dense tables and cards.
- **Reference**: aiadmin.openthome.com (admin dashboard patterns).
- **Anti-references**: Avoid neon gradients, glassmorphism, playful illustrations, generic AI pastel dashboards, or consumer-app aesthetics.
- **Theme**: Support both light and dark modes. Default follows system preference; user can toggle. Dark mode is the existing dominant theme.

### Design Principles
1. **Clarity over decoration**: Every element must earn its place. No purely decorative gradients, glows, or animations.
2. **Information density**: B2B users need to scan lots of data quickly. Use compact tables, consistent spacing, and clear status badges.
3. **Predictable hierarchy**: Same heading sizes, card styles, button roles, and form layouts across all /boss pages.
4. **Mode consistency**: Light and dark modes share the same layout and spacing; only the semantic color tokens change.
5. **Feedback for actions**: Loading, empty, error, and success states are explicit and use the same components everywhere.
6. **Keyboard-friendly desktop UX**: Focus states, accessible form labels, and shortcuts where helpful.
